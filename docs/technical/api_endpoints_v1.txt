API ENDPOINTS GUIDE (v1):
- Authentication: POST /api/v1/auth (requires username/password).
- User Data: GET /api/v1/user/profile (requires Bearer Token).
- Refresh Token: POST /api/v1/auth/refresh.
- System Status: GET /api/v1/health (public endpoint).
- Billing Info: GET /api/v1/billing/history.
All requests must include the header 'Content-Type: application/json'.
